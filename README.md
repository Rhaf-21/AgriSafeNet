# Agricultural Safety Guardrail System

## Project Summary

AgriSafeNet is Agriculture Safety Classifier is a machine learning system that classifies agriculture-related user queries into predefined categories such as safe, unsafe, emergency, and out-of-domain. The model uses TF-IDF for text vectorization and Logistic Regression for classification. The project includes a FastAPI service for prediction APIs, a Streamlit web application for user interaction, and an integrated LLM assistant to provide agricultural guidance based on the classification results.

## Installation

Clone the repository:

```bash
git clone <repository_url>
cd AgriSafeNetProject
```

Create and activate a virtual environment:

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

Linux/macOS:

```bash
source .venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

---

## Run the Project

### Run FastAPI Backend

```bash
uvicorn app:app --reload
```

The API will be available at:

```text
http://127.0.0.1:8000
```

Swagger Documentation:

```text
http://127.0.0.1:8000/docs
```

### Run Streamlit Frontend

```bash
streamlit run streamlit_app.py
```

The web application will be available at:

```text
http://localhost:8501
```

## API Keys & Environment Variables

Create a `.env` file in the project root directory:

```env
OPENAI_API_KEY=your_openai_api_key
```

Example:

```env
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

The application uses this key to generate AI explanations for classification results.

## Known Issues

* The model performance depends on the quality and diversity of the training dataset.
* OpenAI-generated explanations require an active internet connection and valid API key.
* The system is currently optimized for predefined agricultural safety categories.
* Explanations generated by the LLM may vary slightly between requests.
* Additional testing is required for multilingual and unseen inputs.
